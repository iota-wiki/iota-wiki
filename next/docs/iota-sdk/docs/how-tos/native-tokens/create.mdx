The stardust update allows you to create your own [native tokens](https://wiki.iota.org/shimmer/introduction/explanations/what_is_stardust/tokenization#native-tokens)
Native tokens are minted by a so-called Foundry. The Foundry allows you to specify a max supply once and change the circulating supply.

You can also specify some custom metadata in the foundry output or follow a standard like [IRC30](https://wiki.iota.org/tips/tips/TIP-0030/).
To create a Foundry you need an Alias Output which owns it. This Alias Output only needs to be generated once and can own multiple foundries.

## Code Example

<Tabs groupId="language" queryString>
<TabItem value="rust" label="Rust">

The following example will:

1. Instantiate a [`Wallet`](https://docs.rs/iota-sdk/latest/iota_sdk/wallet/core/struct.Wallet.html).
2. Get Alice's [`Account`](https://docs.rs/iota-sdk/latest/iota_sdk/wallet/account/struct.Account.html) which was
   [created in the first guide](../accounts-and-addresses/create-account.mdx).
3. Sync the account with the node to get the latest transactions by calling the [`Account.sync()`](https://docs.rs/iota-sdk/latest/iota_sdk/wallet/account/struct.Account.html#method.sync)
   function.
4. Check the account's balance for an [`Alias Output`](https://wiki.iota.org/shimmer/tips/tips/TIP-0018/#alias-output)
   it can use to create the [native tokens](https://wiki.iota.org/shimmer/tips/tips/TIP-0018/#native-tokens-in-outputs).
   If the account doesn't have any, it will create one.
5. Define the [`CreateNativeTokenParams`](https://docs.rs/iota-sdk/latest/iota_sdk/wallet/struct.CreateNativeTokenParams.html).
6. Create the native token using the [`Account.create_native_token()`](https://docs.rs/iota-sdk/latest/iota_sdk/wallet/account/struct.Account.html#method.create_native_token)
   function.

```rust reference
https://github.com/iotaledger/iota-sdk/blob/develop/sdk/examples/how_tos/native_tokens/create.rs
```
